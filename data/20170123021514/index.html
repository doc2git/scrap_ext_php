----------------------------------------------------------------------------

Open VPN.
网址：http://openvpn.sourceforge.net/
安装：
1.        安装需求：
(1)        Openssl ，非必要但是建议一定要安装（可以用rpm安装）
(2)        TUN/TAP的Driver，在核心2.4.7以前需要另外安装TUN/TAP的套件，但是2.4.7版本以后已经自动加入核心支持中，可以不用安装。
(3)        LZO，实时压缩封包使用，非必要。可以不安装。http://www.oberhumer.com/opensource/lzo/
(4)        Pthread 函示库，用来加快双方交换key的速率。

2.        从上述网址抓取原始档下来安装。
(1)configure –disable-lzo (因为不需要压缩封包，且没装LZO所以要disable)
(2)make
(3)make install
3.        系统设定（设定相关驱动与建立device node）
Linux 2.4 or higher (with integrated TUN/TAP driver):

  (1)  make device node:         mknod /dev/net/tun c 10 200
  (2a) add to /etc/modules.conf: alias char-major-10-200 tun
  (2b) load driver:              modprobe tun
  (3)  enable routing:           echo 1 >; /proc/sys/net/ipv4/ip_forward

  Note that either of steps (2a) or (2b) is sufficient.  While (2a)
  only needs to be done once per install, (2b) needs to be done once
  per reboot.  If you install from RPM (see above) and use the
  openvpn.init script, these steps are taken care of for you.

* Linux 2.2 or Solaris: 要另外安装TUN/TAP驱动。

  You should obtain
  version 1.1 of the TUN/TAP driver from
  http://vtun.sourceforge.net/tun/
  and follow the installation instructions.

4.        规划实际系统
(1)系统概观：
linux187: eth0 : 10.144.43.187
                eth1: 192.168.1.1
                tun : 192.168.2.2
linux142: eth0: 10.144.43.142
                lo:1  : 192.168.3.1
                tun: 192.168.2.1


图一、实验系统架构图
(2)说明
在实验系统中，以linux187为Server、linux142为Client端，这儿所指的Client / Server，是针对OpenSSL再交换Key做认证时的身份，一旦Tunnel建立起来，所有的网络都可以通讯。
   Linux187两张网卡，eth0为界接OA的NIC，其IP address为10.144.43.187，而eth1为界接设备网络的NIC其IP Address为192.168.1.1。eth1所界接的网络为192.168.1.0/24，所有设备的default gateway为192.168.1.1。
   Linux142只需要一张网卡eth0。其OA的IP address为10.144.43.142，另外需要在Linux142上建立一个loopback device lo:1，设定其IP为192.168.3.1/32。
   在这个系统中，Openvpn将建立一个Tunnel，Linux187端的tunnel ip为192.168.2.2，而linux142端的tunnel ip为192.168.2.1。

(3)实际设定
i.首先在Server与Client端建立/etc/openvpn目录。
ii.在Client与Server端先输入下列指令
(i)        modprobe tun
(ii)        echo 1 >;/proc/sys/net/ipv4/ip_forward
iii.因为不启动防火墙，所以NAT、Firwall那一段不提
iv.因为RSA key的交换比较复杂..为了安装速率，本文采取static key的交换方式
v. 在Server端(linux187)产生金钥: oepnvpn –genkey –secret static.key，将这个static.key 放到/etc/openvpn/，同时chmod成600，然后scp到linux142的/etc/openvpn目录。
vi.在linux187的/etc/openvpn目录下，建立几个档案
static-office.conf (给oepnvpn看的)
#使用tun device，ip over ip
dev tun
#设定server端ip与Client端ip，建立tunnel时使用
ifconfig 192.168.2.2 192.168.2.1
#设定Routing 到client端的网络，下面会附上office.up内容
up ./office.up
#使用static.key这个file当成金钥
secret static.key
#使用udp 12345 port当成openvpn预设使用的port
port 12345
#启动tunnel完成后，将openvpn的执行者改为nobody，security issue
user nobody
group nobody
#每15秒ping一次这个tunnel确定tunnel没断
ping 15
#debug信息输出，设定为level 3
verb 3

office.up
#!/bin/bash
#-加入Client端网络的routing，因为实验系统采用loopback ip，
# 所以 netmask为255.255.255.255
route add -net 192.168.3.1 netmask 255.255.255.255 gw $5

在client端linux142上的/etc/openvpn目录
static-home.conf
#使用tunnel dev
dev tun
#设定远程server ip为10.144.43.187
remote 10.144.43.187
#--设定client端ip与server ip，刚好跟linux 187上的ip顺序相反
ifconfig 192.168.2.1 192.168.2.2
#--设定连上server端的routing table
up ./home.up
#--使用static.key这个file当成金钥
secret static.key
#--使用udp/12345当成预设的port
port 12345
#--当tunnel建立完毕，openvpn执行者改成nobody
user nobody

group nobody
#--每15秒ping一次对方，已确定tunnel是正常的
ping 15
#--debug信息，设定为level3
verb 3

home.up
#!/bin/bash
#--加入server端负责的网络 ip address
route add -net 192.168.1.0 netmask 255.255.255.0 gw $5

(4)启动openvpn 服务测试
Server端: openvpn --cd /etc/openvpn --config static-office.conf
Client端: openvpn –cd /etc/openvpn –config static-home.conf
然后在client端(linux142)去ping 192.168.2.2、192.168.3.1、192.168.3.X检查是否正常。在server端(linux187)去ping 192.168.2.1、192.168.3.1，检查是否正常。

然后killall –TERM openvpn或ctrl+c将openvpn停止

(5)写进/etc/rc.local
openvpn-startup.sh(linux142上的)
#!/bin/bash
PATH=$PATH:/usr/local/sbin:/usr/bin:/bin:/usr/bin/:/sbin:/usr/local/bin
openvpn=/usr/local/sbin/openvpn
export PATH
ifconfig lo:1 192.168.3.1 netmask 255.255.255.255 broadcast 192.168.3.1
mobprobe tun >;/dev/null 2>;&1
echo 1 >;/proc/sys/net/ipv4/ip_forward
openvpn --cd /etc/openvpn --daemon --config static-home.conf

openvpn-shutdown.sh(linux142上的)
#!/bin/bash
killall -TERM openvpn

在linux187上的openvpn-shutdown.sh相同，但是openvpn-startup.sh内容不同
#!/bin/bash
PATH=$PATH:/usr/local/sbin:/usr/bin:/bin:/usr/bin/:/sbin:/usr/local/bin
openvpn=/usr/local/sbin/openvpn
export PATH
ifconfig eth1 192.168.1.1 netmask 255.255.255.0 broadcast 192.168.1.255 up
mobprobe tun >;/dev/null 2>;&1
echo 1 >;/proc/sys/net/ipv4/ip_forward
openvpn --cd /etc/openvpn --daemon --config static-office.conf

最后在/etc/rc.local最后面一行加上，Client(linux142)与client(linux187)都要做。
#--start openvpn
/etc/openvpn/openvpn-startup.sh 1>;/dev/null 2>;&1


5.后记：
Openvpn实际上功能相当强大，而本文仅就最简单、且较不安全的static key的方式作介绍，但是实际上如果一般企业要用的话，建议采用document中所建议的RSA方式，且在Server端一定要启动NAT/Firewall相关设定，才能有效的防止网络上的入侵。
